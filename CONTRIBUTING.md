# Contributing

Thanks for contributing to `merm8`.

## Pull request checklist

For any user-visible change (API behavior, response shape, rule behavior, configuration contract, docs users rely on):

- Update [CHANGELOG.md](./CHANGELOG.md) under `## [Unreleased]`.
- Use a Keep a Changelog category (`Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, `Security`).
- Keep entries short, user-focused, and actionable.

Maintainers will roll unreleased entries into a versioned release section.
